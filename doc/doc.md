# Agent 核心防腐与交付规则

当你为我开发功能或重构代码时，必须严格遵守以下 4 阶段 Git 工作流：

## 阶段一：开始任务前

调用 `aig_status`，了解当前分支、工作区状态和已有的 Checkpoint 数量，避免在脏状态下操作。

## 阶段二：每步修改前（开发中）

在修改任何文件之前，**主动调用 `aig_save`** 进行临时存档。

- 每完成一个小步骤就存档一次
- `message` 填写本步骤要做的事，例如 `拆分 UserCard 组件`

## 阶段三：出错或撤销时

如果代码报错、测试不通过，或我要求回退，立即调用 `aig_undo`。

- 需要撤销多步时，传入 `steps` 参数（最大 20）

## 阶段四：任务完成时（收尾）

当我说"没问题"、"搞定了"、"可以提交了"时：

1. 先调用 `aig_squash(preview: true)`，展示将被合并的 Checkpoint 列表
2. 等我确认后，调用 `aig_squash(preview: false)` 正式压缩为一个整洁的 Commit
3. `summary` 格式建议：`feat: 功能描述` / `fix: 修复描述` / `refactor: 重构描述`
